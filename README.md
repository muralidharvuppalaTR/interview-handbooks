# Interview Preparation Handbooks

A curated collection of interview preparation handbooks covering C#/.NET, Backend/Microservices, System Design, Design Patterns, Angular/TypeScript, SQL Server, and Behavioral topics — organized by experience level (Junior to Lead/Architect).

## Handbooks

| Handbook | Topics Covered |
|----------|---------------|
| [C# / .NET Interview Handbook](handbooks/CSharp-DotNet-Interview-Handbook.md) | C#, .NET platform, CLR, async, LINQ, EF Core |
| [C# / .NET Master Handbook](handbooks/CSharp-DotNet-Master-Handbook.md) | Deep-dive: CLR internals, request lifecycle, memory, advanced patterns |
| [Backend & Microservices Handbook](handbooks/DotNet-Backend-Microservices-SystemDesign-Handbook.md) | Backend architecture, microservices, system design |
| [Design Patterns & Cloud Handbook](handbooks/Design-Patterns-Architecture-Cloud-Testing-Handbook.md) | Design patterns, cloud (Azure/AWS), testing strategies |
| [Frontend Angular/TypeScript Handbook](handbooks/Frontend-Angular-TypeScript-Handbook.md) | Angular, TypeScript, RxJS, frontend architecture |
| [SQL Server Handbook](handbooks/SQL-Server-Interview-Handbook.md) | SQL Server, T-SQL, query optimization, indexing |
| [Behavioral Interview Handbook](handbooks/Behavioral-Interview-Handbook.md) | STAR-format behavioral/leadership questions |

## Format

Each handbook follows a **Gated Funnel** structure:

- **Gate 1** — Junior (0-3 years)
- **Gate 2** — Mid (3-7 years)
- **Gate 3** — Senior (7-15 years)
- **Gate 4** — Lead / Architect (15+ years)

Every question includes: **Expected Answer**, **What to listen for**, and **Red flags**.

Dual-purpose: useful for both **candidates studying** and **interviewers evaluating**.

> **Note:** The Master Handbook uses a topic-based "Parts" structure instead of the Gated Funnel format.

## PDFs

PDFs are auto-generated from `.md` files using the VS Code [Markdown PDF](https://marketplace.visualstudio.com/items?itemName=yzane.markdown-pdf) extension and saved in the `pdfs/` folder.

> **Note:** PDFs are git-ignored. After cloning, open any handbook `.md` file and save it to trigger PDF generation. Requires the [Markdown PDF](https://marketplace.visualstudio.com/items?itemName=yzane.markdown-pdf) VS Code extension.
